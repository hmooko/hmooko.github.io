1.5 프로토콜 계층, 서비스 모델

네트워크는 많은 요소들로 구성되어 복잡하기 때문에 네트워크 구조를 조직할 수 있을까라는 생각을 해볼 수 있고 그 구조를 계층화 하는 것이 좋을 수 있다. 계층화하는 이유

  1. 명시적인 구조가 복잡한 시스템의 요소들의 식별과 관계를 분명하게 해줌
    • 논의를 위한 계층화된 참조 모델
  2. 모율화로 시스템의 관리와 갱신이 용이해짐
    • 한 계층의 구현을 변경해도 시스템의 다른 부분에 영향을 주지 않음

인터넷 프로토콜 스택

  • application: 네트워크 응용을 지원
    • FTP, SMTP, HTTP
  • transport: 프로세스간 데이터 전송
    • TCP, UDP
  • network: 출발지에서 도착지까지의 datagram 라우팅
    • IP, routing protocols
  • link: 이웃한 네트워크 요소들간의 데이터 전송
    • Ehternet, 820.111, PPP
  • physical(물리): bits "on the wire"

ISO/OSI 참조 모델

  • 표현(presentation): 응용이 데이터의 의미를 해석하도록 해줌
  • 세션(session): 데이터 교환 시의 동기화(synchronization), checkpointing, 복구(recovery)
  • 인터넷 스택은 이 계층들이 없음

1.6 공격받는 네트워크: 보안

네트워크 보안

인터넷은 원래 상호 신뢰하는 사용자들이 접속된 투명한 네트워크를 기반으로 설계되었기 때문에 보안을 염두에 두지 않았다. 그렇기 때문에 프로토콜 설계자들은 보안문제를 따랍아야 하며 모든 계층에서 보안이 고려되어야 한다.

인터넷을 통해 malware를 침투

  • Malware의 침투 형태
    • virus: 특정 오브젝트를 수신/실행 할 때 자기복제를 통해 감염
    • worm: 수동적으로 수신한 오브젝트가 실행될 때 자기복제를 통해 감염
  • spyware malware는 키보드 입력, 방문한 웹사이트, 업로드 정보 등을 수집해서 특정 사이트로 전송
  • 감염된 호스트들로 botnet이 구성되어 스팸이나 DDoS 공격에 사용

서버나 네트워크 기반 구조를 공격

  • Denial of Service (DoS): 가짜 트래픽으로 자원(서버, 대역폭)을 낭비하게 하여 정상적인 서비스를 어렵게 만든다.
    1. 대상을 선정
    2. 네트워크의 호스트들에 침투
    3. 포섭한 호스트들에서 대상에 패킷을 보냄

1.7 역사

인터넷의 역사: 초기 패킷교환 원리(1961~1972)

  • 1961: Kleinrock – 큐잉 이론으로 패킷 교환의 효율성을 보임
  • 1964: Baran – 군사네트워크에 패킷 교환 적용
  • 1967: ARPAnet 이 ARPA(Advanced Research Projects Agency)에 의해 고안됨
  • 1969: 최초의 ARPAnet 노드가 동작
  • 1972:
    • ARPAnet 공개 데모
    • NCP (Network Control Protocol) 최초의 호스트간 프로토콜
    • 최초의 e-mail 프로그램
    • ARPAnet에 15 개의 노드 존재

인터넷의 역사: 인터네트워킹, 새로운 독점 네트워크(1972~1980)

  • 1970: 하와이의 ALOHAnet 위성 네트워크

  • 1974: Cerf and Kahn – 네트워크 상호연결을 위한 아키텍처: 오늘날의 인터넷 아키텍처를 정의

    • minimalism, autonomy – 네트워크 상호연결을 위해 내부변경을 요구하면 안됨
    • best effort 서비스 모델
    • stateless routers
    • decentralized control

  • 1976: Xerox PARC의 Ethernet

  • 70년대 후반: 독점 아키텍처들: DECnet, SNA, XNA

  • 70년대 후반: 고정길이 패킷의 스위칭 (ATM 의 전조)

  • 1979: ARPAnet에 200개의 노드

인터넷의 역사: 새로운 프로토콜, 네트워크의 확산(1980~1990)

새 국가 네트워크(CSnet, BITnet, NSFnet, Minitel)가 생겨나고 100,000 대의 호스트가 네트워크에 연결되었다.

  • 1982: SMTP e-mail 프로토콜 정의
  • 1983: TCP/IP 채택
  • 1983: 이름-IP주소 변환을 위한 DNS 정의
  • 1985: ftp 프로토콜 정의
  • 1988: TCP 혼잡 제어

인터넷 역사: 상업화, 웹, 새 응용들(1990, 2000's)

  • 1990년대 초: ARPAnet 퇴역
  • 1991: NSFnet 의 상업화 제한 해제(1995년에 퇴역)
  • 1990년대 초: Web
    • hypertext [Bush 1945, Nelson 1960’s]
    • HTML, HTTP: Berners-Lee
    • 1994: Mosaic, 후에 Netscape
    • 1990년대 후반: 웹의 상업화
  • 1990대후반 – 2000년대
    • 더 많은 killer app들: instant messaging, P2P 파일 공유
    • 보안 문제가 대두됨
    • 약 5천만 대의 호스트, 1억명 이상의 사용자
    • backbone link가 Gbps급으로

인터넷 역사(2005~현재)

  • 50억대의 장비가 인터넷에 연결됨 (2016)
    • 스마트폰과 태블릿 등
  • 광대역 접속의 적극적 배치
  • 고속 무선 접속의 증가
  • online social network 등장:
    • Facebook: ~ 10억명의 사용자
  • 서비스 제공자들(Google, Microsoft) 자체 네트워크 구성
    • 인터넷을 우회하여 검색, 비디오, 메일 등을 즉시 접속하도록 함
  • 전자상거래, 대학, 기업 등이 “클라우드”에서 서비스 운영 (e.g., Amazon EC2)
